# Эксперимент

Хочу поставить эксперимент про разработку программного обеспечения. В эксперименте будем участвовать мы с тобой (то есть пользователь и ИИ-ассистент). На каждом этапе мы будем выполнять разные роли.

В данном проекте наша роль - наблюдателей и незримых помощников всего процесса. Мы будем формулировать и ставить задачи для последующих этапов. Мы - те, кто ставит эксперимент и анализирует результаты процессов, проходящих на тех этапах. Также ты будешь составлять инструкции для ИИ-ассистента всех этапов. В конце эксперимента мы обсудим результаты эксперимента и ты поможешь сформулировать выводы.

**Входные документы:**

- [Конспект лекции по Инженерии требований](requirements-engineering-summary.pdf);
- [Issue: требования, тикеты и Merge Request](REQUIREMENT.md) — копия файла из другого проекта, ради которого проводится данных эксперимент. В документе есть ссылки на документацию, ссылки - не рабочие, текст самодостаточен и без дополнительных файлов.

**Цель эксперимента:** предложить **простой сценарий обеспечения наличия каждого свойства требования** (кроме свойства Приоритизация). Реализация сценария не обязательно должна быть автоматической. Цель эксперимента совпадает с целью другого проекта, см. в [REQUIREMENT.md](REQUIREMENT.md).

## Этапы эксперимента

Планируется, что я (пользователь) буду выражать своё мнение только как экспериментатор, то есть только в этом проекте. В ходе выполнения эксперимента я буду лишь подготавливать входные данные, запуская процесс указывать ИИ-ассистенту на входные данные, следовать указаниям ИИ-ассистента, отвечать на вопросы ИИ-ассистента.

Выполнение этапа эксперимента может быть многоитерационным. Если итерация будет признана неуспешной, мы вернёмся в этот проект, чтобы исправить инструкции для ИИ-ассистента, чтобы попробовать выполнить неудавшийся этап ещё раз.

Если этап должен быть проведён в несколько шагов, то инструкции ИИ-ассистенту должны содержать принцип HITL для оценки результатов выполнения шага.

К проекту подключен MCP-сервер `gitlab-ai-helper` для наблюдения за результатами этапа "Оформление требований". Ещё одной дополнительной целью проекта является добавление или модификация набора инструментария ИИ-агента, использующего данный MCP-сервер для удобства его использования на последующих этапах. У тебя есть доступ на чтение.
