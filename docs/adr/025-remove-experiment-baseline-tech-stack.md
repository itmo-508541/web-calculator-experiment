# ADR-025: Удаление базового `tech-stack.md` и инструкции инвентаризации из эксперимента

## Статус

Принято; заменяет [ADR-015](015-lead-dev-tech-stack-handoff.md) в части «файлы остаются историческим артефактом в репозитории эксперимента». Сохраняет: вход ИИ Ведущего = Issue + репо Разработчика; выход = `tech-stack.md` у Ведущего → снимок у PM в `docs/`.

Удалены [`tech-stack.md`](../../tech-stack.md) и [`docs/tech-stack-inventory-INSTRUCTIONS.md`](../tech-stack-inventory-INSTRUCTIONS.md); обновлён [README.md](../../README.md) — 2026-07-20.

## Дата

2026-07-20

## Контекст

В эксперименте лежали инвентаризация корпоративного каркаса (`tech-stack.md`) и инструкция её сбора (`docs/tech-stack-inventory-INSTRUCTIONS.md`). С [ADR-015](015-lead-dev-tech-stack-handoff.md) они **не** вход этапа 4: ИИ Ведущего собирает краткий стек из репозитория Разработчика. Актуальные файлы живут там, где используются: `senior-developer/tech-stack.md`, `product-manager/docs/tech-stack.md`.

Согласовано с экспериментатором: убрать дубликаты из experimentator (как ранее `docs/developer/`).

## Решение

1. Удалить из репозитория эксперимента корневой `tech-stack.md` (базовый каркас) и `docs/tech-stack-inventory-INSTRUCTIONS.md`.
2. Не восстанавливать их как обязательный артефакт наблюдения: для процесса достаточно файлов в директориях ролей и ссылок в README/ADR.
3. Старые ADR (009, 014, 015), ссылающиеся на удалённые пути, **не переписывать** — это история; актуальный locus — настоящий ADR и рабочие каталоги ролей.

## Обоснование

- Исполняемый процесс не читает эти файлы.
- Инвентаризация «пустого приложения» избыточна рядом с уже переданным продуктовым `tech-stack.md`.
- Единый принцип: инструкции и артефакты — в директориях использования.

## Рассмотренные альтернативы

| Альтернатива | Почему отклонена |
| --- | --- |
| Оставить как «исторический артефакт» в git (ADR-015) | Путает с актуальным стеком; не нужна наблюдателю |
| Перенести в `docs/archive/` | Лишняя полка без читателей |

## Последствия

- README этапа 4 не ссылается на корпоративный `tech-stack.md` эксперимента.
- Источник правды о стеке продукта для PM — `product-manager/docs/tech-stack.md` (копия от Ведущего).
