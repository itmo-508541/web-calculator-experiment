# ADR-023: Инструкции этапа 5 только в директории Разработчика

## Статус

Принято; заменяет [ADR-022](022-stage5-cursor-commands-task-adr.md) в части **источника текстов** команд (`docs/developer/` в эксперименте + зеркало в продукте). Сохраняет из ADR-022: команды `/task` и `/adr`, порядок запуска, хронологические ADR продукта, смысл статуса **Принято**; из ADR-021 — роли, Code Review, сдача PM.

Удалены [`docs/developer/task.md`](../developer/task.md), [`docs/developer/adr.md`](../developer/adr.md), [`docs/developer/INSTRUCTIONS.md`](../developer/INSTRUCTIONS.md); обновлён [README.md](../../README.md) (этап 5) — 2026-07-20.

## Дата

2026-07-20

## Контекст

После ADR-022 полные тексты `/task` и `/adr` дублировались: зеркало в `experimentator/docs/developer/` и исполняемые файлы в `developer/.cursor/commands/`. Двойная правка утомляла и расходилась. Для наблюдения за экспериментом достаточно ссылки на место использования; полный текст нужен ИИ Разработчика в его окружении.

Согласовано с экспериментатором: убрать каталог `docs/developer/` в эксперименте; все инструкции и команды этапа 5 живут там, где используются.

## Решение

### 1. Единственный источник команд этапа 5

| Команда | Файл |
| --- | --- |
| `/task` | [`/home/covex/Calculator/developer/.cursor/commands/task.md`](/home/covex/Calculator/developer/.cursor/commands/task.md) |
| `/adr` | [`/home/covex/Calculator/developer/.cursor/commands/adr.md`](/home/covex/Calculator/developer/.cursor/commands/adr.md) |

Отдельного зеркала и указателя `INSTRUCTIONS.md` в репозитории эксперимента **нет**.

### 2. Роль эксперимента

В [README.md](../../README.md) (этап 5) — краткое описание процесса и **ссылки** на файлы команд в директории Разработчика. Правки формулировок команд — только в `.cursor/commands/` продукта; существенные решения о процессе по-прежнему фиксируются ADR эксперимента.

### 3. Принцип для других этапов

Инструкции и команды держать в рабочих директориях ролей, где они запускаются (как `account-manager/`, `business-analyst/`, `product-manager/`, `senior-developer/`, `developer/.cursor/commands/`), без обязательной полной копии в `experimentator/docs/`.

## Обоснование

- Один файл на команду исключает рассинхрон.
- Исполняемый контекст Cursor — репозиторий продукта; эксперименту достаточно навигации и ADR.
- Согласуется с принципом «инструкция рядом с исполнением».

## Рассмотренные альтернативы

| Альтернатива | Почему отклонена |
| --- | --- |
| Оставить зеркало в `docs/developer/` (ADR-022) | Двойная поддержка без выигрыша для исполнения |
| Только указатель в эксперименте без полных текстов | Лишний файл; достаточно README |
| Вынести команды в experimentator и открывать из продукта | Ломает удобный `/task` в workspace Разработчика |

## Последствия

- Каталог `experimentator/docs/developer/` удалён.
- ADR-022 остаётся историей введения `/task` и `/adr`; актуальный locus файлов — настоящий ADR.
- Наблюдение текстов команд этапа 5 — из директории Разработчика.
