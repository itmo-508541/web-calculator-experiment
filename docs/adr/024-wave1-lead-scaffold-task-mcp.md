# ADR-024: Волна 1 — процесс Ведущего (каркас), `/task` без CLI GitLab/MR

## Статус

Принято; дополняет [ADR-015](015-lead-dev-tech-stack-handoff.md) и [ADR-014](014-lead-dev-module-recommendations.md) процессом **обсуждения состава каркаса** с ИИ до/во время доведения репозитория; уточняет [ADR-021](021-stage5-dev-mr-review-pm-handoff.md) / [ADR-022](022-stage5-cursor-commands-task-adr.md) в части создания MR и доступа к GitLab из `/task`. Сохраняет: низкоуровневый стек обязателен; рекомендации модулей Разработчику — ориентир; Code Review и сдача PM; команды в продукте ([ADR-023](023-stage5-commands-live-in-product.md)).

Обновлены [`senior-developer/INSTRUCTIONS.md`](/home/covex/Calculator/senior-developer/INSTRUCTIONS.md), [`developer/.cursor/commands/task.md`](/home/covex/Calculator/developer/.cursor/commands/task.md), [FINDINGS.md](../../FINDINGS.md) (**F-011**), [README.md](../../README.md) — 2026-07-20.

## Дата

2026-07-20

## Контекст

Первая волна задач Разработчика **выполнена** (реализация, ADR продукта, MR, ревью, сдача). Задержка волны: Ведущий сначала скопировал/подрезал каркас (в т.ч. убрал MySQL/database), недостаточно сверившись с Issue PM — не подготовил модули для E2E; ИИ Ведущего по ADR-015 в основном **фиксировал итог** в `tech-stack.md` и не помогал разобрать «что добавить/убрать относительно задачи».

На этапе 5 ИИ Разработчика тянулся к `glab`/`gh`; модели по-разному создавали MR (один вариант пришлось переделывать). Нужна жёсткая граница: Issue — MCP; MR — только руками Пользователя.

Для GitLab AI Helper (отдельно): нецелесообразно догонять полный CLI GitLab — см. **F-011** в FINDINGS.

## Решение

### 1. Исход волны 1

Первая волна задач корпуса закрыта процессом этапа 5 (`/task` → `/adr` → MR Пользователем → Code Review → accept → сдача PM). Дальше — следующая волна по плану PM.

### 2. Процесс Ведущего при подготовке каркаса

Типичный порядок (осознанно допускает «сначала копия, потом сверка с Issue»):

1. Беглое ознакомление с задачей / доменом.
2. **Принципиальный выбор стека** — только Ведущий (не ИИ).
3. Копирование своих наработок в директорию Разработчика.
4. По Issue от PM — **обсуждение с ИИ**: что убрать / что добавить относительно текущего репо и классов потребностей задачи (в т.ч. QA/E2E); чеклист пробелов, HITL.
5. Доведение репозитория до требуемого вида — Ведущий **с ИИ или без**.
6. Фиксация краткого `tech-stack.md` для PM (как в ADR-015): факты «что есть сейчас»; рекомендации модулей Разработчику — по-прежнему только от Ведущего, опционально.

ИИ Ведущего **не** выбирает принципиальный стек и **не** проектирует нестандартный каркас «с нуля». Помогает **сверять** Issue с фактом репозитория и предлагать изменения состава для обсуждения — иначе на чужом каркасе ИИ «чудит».

### 3. Дельта `/task` (Разработчик)

Уже в [`task.md`](/home/covex/Calculator/developer/.cursor/commands/task.md):

- GitLab / Issue — **только MCP** (`gitlab-ai-helper`); **запрещены** `glab`, `gh` и обход MCP.
- **Не** создавать Merge Request и **не** помогать оформлять его в GitLab — только Пользователь.
- После `/adr` Пользователь сам открывает MR; ИИ разбирает верификацию MR и Code Review **только если** Пользователь передал текст; готовит пакет сдачи PM после accept.

### 4. FINDINGS

Зафиксировать **F-011**: не догонять `glab`/`gh`; оставить в Helper/MCP то, чего нет в GitLab (сейчас — история изменений Issue); перспектива — требования в Wiki проекта.

## Обоснование

- Пробел E2E — следствие слабой сверки каркаса с Issue, а не только «плохой копии»; шаг обсуждения с ИИ закрывает этот разрыв без передачи выбора стека ИИ.
- Разные модели нестабильно создают MR через CLI — ручное создание предсказуемее для вайб-кодера.
- MCP — контракт эксперимента с Helper; `glab`/`gh` маскируют пробелы продукта и плодят расхождения.

## Рассмотренные альтернативы

| Альтернатива | Почему отклонена |
| --- | --- |
| Оставить ИИ Ведущего только на финальном `tech-stack.md` (ADR-015) | Не предотвращает пропуск модулей (E2E) до передачи PM |
| ИИ сам выбирает и собирает каркас | Нестандартный корпоративный каркас; экспериментатор отвергает |
| Разрешить `glab`/`gh` для MR | Нестабильность между моделями; обход MCP |
| ИИ всегда создаёт MR | Явно отвергнуто после прогона волны 1 |
