# ADR-013: Идентификаторы задач в системе хранения — заголовок `TASK-…` и ссылки `#N`

## Статус

Принято; уточняет [ADR-007](007-pm-task-trace-roles.md) (форма трассировки **в теле Issue**); зеркалит [ADR-012](012-ba-appendix-b-storage-refs.md) для Продукт-менеджера; правки [`product-manager/INSTRUCTIONS.md`](/home/covex/Calculator/product-manager/INSTRUCTIONS.md) внесены (2026-07-15). Уже созданные Issue PM **не** нормализуются наблюдателями эксперимента — это делает Пользователь с Продукт-менеджером.

## Дата

2026-07-15

## Контекст

У BA (ADR-012): title Issue = `REQ-NNN <Краткий заголовок>`; в теле после переноса отсылки на другие требования — только `#N`. Приложения B у PM нет; рабочий текст — `developer-tasks.md`.

Экспериментатор: во внутренних файлах PM id нужны; **в системе хранения** в теле задачи не нужны ни `REQ-…`, ни `TASK-…` — только в **заголовке** задачи (`TASK-…` в начале title). `TASK-…` в теле допустимы лишь пока задача ещё не в GitLab. Трассировка к требованиям не нарушается, потому что title требования уже начинается с `REQ-…`.

## Решение

1. **Title Issue задачи:** `TASK-NNN <Краткий заголовок>` (пробел, без обязательного тире), по аналогии с `REQ-NNN …` у BA. Сборка из `tasks-list.md` (колонки «Задача» + «Заголовок»).
2. **Тело Issue после переноса / нормализации:** только `#<номер>` на требования (`requirements-list.md`) и на другие задачи (`tasks-list.md`); роли **выполнить** / **не нарушить** сохраняются; строк `REQ-…` и `TASK-…` в теле нет.
3. **Черновик до переноса** в `developer-tasks.md`: допустимы `TASK-…` и при необходимости `REQ-… #N`.
4. **После** сообщения номеров Пользователем: обновить `tasks-list.md`, нормализовать тексты для Issue, HITL; правку GitLab — Пользователь или PM по явному поручению.
5. Локальные артефакты (`task-plan.md`, заголовки секций, колонка «Задача») по-прежнему используют `TASK-…`.

## Обоснование

- Единые правила BA и PM для GitLab: внутренний id в title, нативные `#N` в теле.
- Трассировка через title связанных Issue (`REQ-…` / `TASK-…`) + `#N` в теле достаточна без дублирования внутренних id в описании.
- Нормализация после переноса не блокирует черновик волны.

## Рассмотренные альтернативы

| Альтернатива | Почему отклонена |
| --- | --- |
| Оставлять в теле `REQ-001 #25` и `TASK-001` | Явно отвергнуто экспериментатором для системы хранения |
| Убрать `TASK-…` из title Issue | Теряется стабильный id в списке GitLab |
| Менять уже залитые Issue силами экспериментатора | Роль наблюдателей; нормализация — с PM |

## Последствия

- ADR-007 остаётся в силе по **ролям** ссылок; форма ссылки в Issue — `#N` + роль (не обязательная пара `REQ-… #N` в теле).
- Инструкция PM описывает title, нормализацию и отличие локальных файлов от текста Issue.
