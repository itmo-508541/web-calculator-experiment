## Анализ требований

-  если Выявление требований – расходящийся процесс, цель которого собрать как можно больше данных
-  то Анализ требований – сходящийся процесс:
  -  уточняет данные
  -  устанавливает приоритеты
  -  структурирует информацию
-  Результат анализа – спецификация требований
-  Цель этапа – таким образом переработать, уточнить, дополнить, переформулировать требования, чтобы они стали удовлетворять 8 характеристикам требований
  -  Корректность
  -  Однозначность
  -  Полнота
  -  Непротиворечивость
  -  Приоритизация
  -  Проверяемость
  -  Модифицируемость
  -  Отслеживаемость
### Техника анализа требований: Уточнение

-  Каждое требование должно быть максимально полным, корректным и однозначным
-  Уточнение достигается путем повторных встреч с заинтересованными лицами
-  Не должно появляться много новых требований – иначе следует вернуться к выявлению
-  На этапе уточнения требования должны быть описаны количественно, а не качественно, как на этапе выявления
-  Для обеспечения свойства проверяемости, требования должны быть описаны количественно, а не качественно
-  Для каждого переработанного требования должен быть понятен метод проверки его выполнимости
### Техника анализа требований: Устранение противоречий

-  Для обеспечения свойства непротиворечивость список требований анализируется на наличие явных и неявных скрытых противоречий
-  Найденные противоречивые требования перерабатываются с участием заказчика
### Техника анализа требований: Приоритизация

-  Для обеспечения свойства приоритизируемость необходимо провести приоритизацию требований
-  Необходимо отсортировать требования по важности и срочности
-  Должны участвовать все заинтересованные лица проекта:
  -  заказчики
  -  пользователи
  -  исполнители
-  Все требования не могут быть основными и срочными
-  Приоритеты могут изменяться по мере развития проекта, если методология разработки позволяет это сделать
-  Каждое требование относится к какой-либо качественной категории по важности:
  -  Высокая, средняя, низкая
  -  Обязан, должен бы, мог бы
  -  Основной, полезный, желаемый
-  Каждое требование относится к какой-либо качественной категории по срочности
  -  Прямо сейчас, чуть позже, когда-нибудь
  -  Срочно, чуть позже, потом
