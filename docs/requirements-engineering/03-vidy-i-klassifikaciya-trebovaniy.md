## Дополнительные характеристики требований

-  Единичность
  -  требование описывает только одну и только одну вещь
-  Атомарность (неделимость)
  -  требование не может быть разбито на ряд более детальных требований без потери завершенности
-  Актуальность
  -  свойство требования не быть устаревшим
  -  требование не стало устаревшим с течением времени
-  Выполнимость
  -  Требование может быть реализовано в пределах проекта
## Виды требований:

-  Функциональные требования
  -  Бизнес-требования
  -  Пользовательские требования
  -  Собственно функциональные требования (бизнес- и пользовательские требования) переработанные в спецификацию требования
  -  Отвечают на вопрос какой функциональностью должно обладать ПО
-  Нефункциональные требования
  -  Ограничения
  -  Требования к качеству
## Функциональные требования

-  Бизнес-требования
  -  Формулируются заказчиками
  -  Описывают цели, которые требуется достичь с данной системой
  -  Описывают назначение ПО (описываются в документе видении и границах проекта)
-  Требования пользователей
  -  Какие задачи можно решить с помощью системы
  -  Отражают взгляд пользователя
  -  Какие задачи пользователь может решить используя это ПО
-  Собственно функциональные требования
  -  Определяются функциональность, которую необходимо реализовать
## Нефункциональные требования

-  Отвечают на вопрос “как это должно быть сделано”
-  Характеристики качества
  -  Требования к надежности
  -  Требования к совместимости
  -  Требования к эффективности
    -  память, скорость, ресурсы
  -  Требования к гибкости
  -  Требования к эргономике
  -  Не добавляют никакой функциональности к системе, говорят о том как система функционирует
-  Ограничения
  -  Ограничения на программные интерфейсы, в том числе к внешним системам
  -  Требования к применяемому оборудованию и ПО
  -  Соответствия стандартам и правилам
  -  Предопределенные архитектурные решения (*)
    -  обычно это является результатом проектирования
    -  ограничение применяют, если нужно встроиться в какую-то систему
    -  например, при разработке плагина для VS Code -> type script, и нужно использовать спец.протокол
  -  Бюджет
  -  Сроки
